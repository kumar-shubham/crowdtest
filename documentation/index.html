Upload all the files into your website's FTP folder. You can use an FTP client like Filezilla.

CHMOD FOLDERS & FILES

The following folders must have their permissions changed via FTP (CHMOD).

This will make them writeable. Normally you simply right click the specified file/folder on the server side of you FTP Client and you select 'properties' you will then have an option to set the permission levels that you want. 

 Typically for Folder you set the permission level to 777 or 755 (depending on your web host's settings).

      If you have set to 777  and you find you are getting 500 SERVER ERROR then you must CHMOD back to 755.

      For file, the permission level needed is normally 666

I) Sprint - top

All the story or issue included inside the sprint should becompleted within that time frame.The SUCCESS story is all about breaking bigger tasks into smaller ones and tracking it till the delivery.

K) Sprint Release Burndown Chart - top

The sprint release burn down chart is a one stop solution to understand whether the project is on track or not.


FIG.15

The red line is the estimated interpretation of the sprint. The green line shows the actual interpretation.In the above chart we can see the green line goes straight this means either there is either a requirement gap or the product owner could not estimate the sprint. Ideally the green and red line should again meet at the end to denote the work has been completed on time.

If the green line is above the red line at any point that means the time taken is to deliver is more than estimated, whereas if it is below then it denoted the team is well ahead of the estimated time.
